## Garanzie sicurezza

- Storage paths: `repository/{ownerUid}/...` — solo owner autenticato può leggere/scrivere
- Firestore `programs/{programId}`: solo owner può leggere/scrivere (isOwner check)
- `.pool.md` escluso da ZIP (test automatico — securityReview.test.ts)
- `.pool.md` non renderizzato in UI (non esposto via lesson storageRef)
- ZIP/PDF non salvati in Storage (nessuna chiamata uploadBytes/uploadString)
- questionIndex non contiene testo domande (validato in M1-A, struttura solo conteggi)

## Limiti residui

- Test E2E browser reali non implementati (Playwright non configurato per M1-G; smoke test con mocks coprono i flussi critici)
- jsPDF in jsdom non può generare PDF reale (smoke test verifica solo che la funzione sia invocabile)
- ZIP non viene effettivamente creato in test (jszip mockato; test verifica logica di inclusione/esclusione file)
- Emulatori Firebase non avviati in CI (test:rules usa firebase-admin con emulator; richiede emulatore locale)

## Rischi tecnici

- Se Storage rules vengono modificate senza aggiornare i test, la garanzia owner-scoped potrebbe non essere rilevata
- Bundle size con jsPDF è grande (Vite warning pre-esistente); lazy import già usato
